# ⛳ 富加ゴルフ レッスン予約システム 要件定義書

## 1. プロジェクト概要
富加ゴルフにおける1名体制のレッスン業務を最適化する専用Webアプリケーション。
LINEを起点としたパスワードレスな顧客体験、Googleカレンダー同期による管理者負担の軽減、AIと動画を用いた独自の「復習帳」機能による顧客満足度の向上を目的とする。

- **プロジェクトルートフォルダ**: `tomika-golf-lesson`

## 2. システムアーキテクチャ・技術スタック
- **フロントエンド**: Next.js (App Router), Tailwind CSS (テーマカラー: 深い緑 `#1A6618`)
- **バックエンド・DB・ストレージ**: Supabase (Row Level Securityを用いたアクセス制御を実装)
- **ホスティング**: Vercel
- **外部API連携**:
  - LINE LIFF (パスワードレス自動ログイン用)
  - LINE Messaging API (各種通知用)
  - Google Calendar API (予約枠の在庫管理用)
  - Gemini API (音声認識からのカルテ生成、およびチャットアシスタント用)

## 3. 画面構成・静的ページ
- **LP・情報ページ**: 「富加ゴルフのレッスンコンセプト」「講師紹介」ページ。ここからLINE公式アカウントの友だち追加へ誘導。
- **アプリケーション画面**: LINEのリッチメニューからLIFFブラウザ経由でアクセスする「予約画面」「マイページ」など。

## 4. 予約・スケジュール管理ロジック（最重要仕様）

### 4.1. Googleカレンダーの自動スライシング処理
- 管理者がGoogleカレンダー上で「9:00〜13:00」のようにブロックで予定（レッスン稼働時間）を登録する。
- システムはこれを読み込み、**「毎時00分開始、50分間（10分インターバル）」**の独立した予約枠として分割（スライス）し、フロントエンドの予約画面に表示する。
  - *(例: 9:00〜9:50 / 10:00〜10:50 / 11:00〜11:50 / 12:00〜12:50)*

### 4.2. レッスン種別と事前ヒアリング設定
- **マンツーマンレッスン**:
  - 事前ヒアリング：希望課題（ボタン選択＋自由記述）
  - **限定オプション**: 「芝（＋1,000円）」「バンカー（＋1,000円）」を別々のボタンで選択可能。
- **グループレッスン**:
  - 事前ヒアリング：希望課題（ボタン選択＋自由記述）のみ。オプションの選択は不可。

## 5. ビジネスルール（体験制限・チケット管理）

### 5.1. 初回体験（トライアル）の制限
- マンツーマン・グループともに、初回は「体験特別価格」の扱いとなる。
- システム制御：**「1回目のレッスンが未受講（完了ステータスになっていない）状態では、2回目の予約を取得不可」**とする。

### 5.2. チケット管理仕様（マンツーマンとグループは別枠管理）
データベース上で「マンツーマン用チケット」と「グループ用チケット」の残数を完全に独立して管理する。

- **マンツーマンの予約ルール**:
  - 初回体験完了後、2回目以降は先の予約（3回目、4回目など）も自由に取得可能。
- **グループの予約ルール**:
  - 初回体験完了後、2回目の予約（通常料金1回目）はそのまま取得可能。
  - **3回目以降の予約は「グループ用チケット」の残数が1以上ないと予約不可**とする。
- **チケットの付与・消費（手動運用・自動消費禁止）**:
  - チケットは日時の経過で自動消費させてはならない。お客様が来店・受講後、管理画面から「受講完了」処理を行うことで残数が1減る仕様とする。
  - 管理画面から、任意のユーザーに対して各種チケットの「付与」および「残数修正」が行えること（イレギュラー対応・不正防止のため）。

## 6. キャンセルポリシーのシステム制御
- **前日までのキャンセル**: マイページ上のボタンからお客様自身でキャンセル可能（チケット残数は変動しない）。
- **当日のキャンセル**: マイページ上のキャンセルボタンを非活性化（クリック不可）。「本日のキャンセルはお電話、または公式LINEにて直接ご連絡ください」という案内文を表示し、システム上での直前キャンセルを防ぐ。

## 7. 顧客向け機能（マイページ・アフターフォロー）

### 7.1. 日付ベースの「復習帳」機能
- タイムライン形式ではなく、レッスン実施日ごとの**「日付独立型ページ（例：4月22日のレッスン記録）」**を動的に生成する。
- 各ページには以下が含まれる。
  - 講師がアップロードした専用スイング動画（Supabase Storage経由、本人以外閲覧不可）。
  - AI生成カルテ（①良かった点、②直すべき点・原因、③宿題 の3段構成）。

### 7.2. 専属AIアシスタント機能
- マイページ内にチャットUIを実装。
- 汎用的な会話ではなく、**「そのユーザーの過去の復習帳（カルテ内容）」のみをコンテキストとしてGemini APIに渡す**。
- ユーザーの質問に対し、回答とともに「該当する過去のレッスン記録（日付ページ）」へのリンクを提示し、復習を促す。

## 8. 管理者向け機能
- **予約・ユーザー管理**: 予約一覧の閲覧、チケット残数の手動調整、受講完了処理。
- **個別キャンセル（代理キャンセル）**: お客様から電話等で連絡があった際、管理者がシステム上で特定の予約を個別にキャンセルし、枠を空ける機能。
- **緊急一括キャンセル**: 特定の日付を指定し、該当日の全予約をキャンセル＋対象者へLINEでお詫びメッセージを一斉送信する機能。
- **カルテ・動画作成**: 音声入力（Web Speech API等）をGemini APIに送信してテキスト化・修正し、動画アップロードと同時に「復習帳ページ」を生成する機能。※ブラウザのマイク制限回避のため、管理画面はPCでの運用を前提とする。

## 9. 通知連携
- **予約確定時**: LINE Messaging APIによる即時通知。
- **個別・一括キャンセル時**: 管理者側でキャンセル処理を実行した際、該当ユーザーへLINE通知。
- **レッスン前日**: CRONジョブ等によるLINEリマインド通知。
